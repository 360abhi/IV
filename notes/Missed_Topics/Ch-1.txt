üîπ 2Ô∏è‚É£ Playwright ‚Äî sync_api, sync_playwright, p
Code you write:

What is what:
Name	Type
playwright	package
sync_api	module

p	Playwright object
Simplified internal model:
class Playwright:
    def __enter__(self): return self
    def chromium(self): return BrowserType()

class BrowserType:
    def launch(self): return Browser()

class Browser:
    def new_context(self): return BrowserContext(self)

class BrowserContext:
    def __init__(self, browser): self.browser = browser
    def new_page(self): return Page(self)

class Page:
    def __init__(self, context): self.context = context

Composition:
Playwright ‚Üí BrowserType ‚Üí Browser ‚Üí Context ‚Üí Page


Not inheritance ‚Äî composition.

Playwright: Strict Mode, locator() vs page.$
1Ô∏è‚É£ What is a Locator?

A Locator is Playwright‚Äôs element reference.

page.locator("#submit")


It does not immediately find the element ‚Äî it describes how to find it.

2Ô∏è‚É£ What is Strict Mode?

Strict mode means:

A locator must match exactly one element.

If it matches:

0 ‚Üí error

More than 1 ‚Üí error

This prevents ambiguous actions.

Example:
page.locator("//button").click()


If there are 3 buttons ‚Üí ‚ùå error

Fix:

page.locator("//button").first.click()


or

page.locator("//button", has_text="Submit").click()

3Ô∏è‚É£ page.locator() vs page.$()
Feature	page.locator	page.$
Returns	Locator	ElementHandle
Waits	Yes	No
Strict mode	Yes	No
Live reference	Yes	No
Retry	Yes	No
Recommended	Yes	No
4Ô∏è‚É£ page.$() example
el = page.$("#submit")
el.click()


If element not loaded ‚Üí None / fail.

5Ô∏è‚É£ page.locator() example

Playwright waits until:

element exists

6Ô∏è‚É£ Why strict mode exists

Prevents clicking wrong element

Prevents flaky tests

Forces good selectors
